Something is Wrong with Hexo-NexT-Music😱

发表于 2016-08-19   |   分类于 爬出坑后写清单   |     |   阅读次数

写给2016年6月后的博主

岛主搭建博客时遇到一个小问题，他怀疑是markdown编辑器的问题，所以发邮件予我拜托我试一下，正巧我吃完中饭休息一会儿，对照着他找的一份很棒的经验帖，贴了代码本地预览立即听到了音乐🎵，觉得好有成就感，但是部署到github page上后在网页端预览时，音乐无法显示。



大热天的这盆凉水把午后的倦意一下子冲走了。
不午睡的人生就干点活呗~
以下算是清单的一次小实践😋
感谢岛主提出的好问题~

问题描述

按照hexo创建博客（附加插入音乐）中的指导，插入的代码：

<iframe frameborder="no" border="0" marginwidth="0" marginheight="0" width=330 height=86 src="http://music.163.com/outchain/player?type=2&id=424262521&auto=1&height=66"></iframe>

发现，本地运行ok，音乐很好听
BUT, 网页端无法显示👇



排除插件问题

不过，这个时候还有个可能是因为插件的问题，毕竟网易云音乐有这样一段提醒：



所以这个时候有个很自然的假设：

因为上述代码是iframe插件，本地预览没有问题，但是同步后网页版无法显示。flash插件的代码应该更保险吧~

所以继续试验，在上述基础上插入下列代码：

<embed src="http://music.163.com/style/swf/widget.swf?sid=424262521&type=2&auto=1&width=320&height=66" width="340" height="86" allowNetworking="all"></embed>

谷歌浏览器上和Safiri游览器上它们的表现本质上是一致的：

这两个插件在本地都可以正常显示，但是flash插件的速度明显要快一些。同时Safiri浏览器不会默认直接播放flash的音乐（它要省电😂）

但是网页端，他们都失败了。
至此，排除插件问题。



似乎找到问题所在

在网上搜了下这个问题，这个链接中的回答还挺靠谱的。灰常有可能是以下原因：

  • 网易提供的外部链接都是http://打头的
  • github的博客是https://打头的，所以不行
  • 本地预览链接为http://localhost:4000/, 是http://打头的，所以米有问题~

确定问题所在

恩，然后我找了一些用NexT主题引进音乐的博客，进一步看了下这个问题：

博客一：http://www.qiuchengjia.cn/
NexT主题，一进去就有音乐~绑定了自己的域名，写了篇灰常简洁的嵌入音乐经验帖，之所以这么简单应该就是多亏绑定域名上述问题不存在了吧~😏

博客二：http://login926.github.io/2016/07/20/HexoMedia/
NexT主题，为了插入音乐，博文以及评论描述了这段坎坷的旅程，有一句话很关键APlayer 与 DPlayer 均支持https。

不过，我也很好奇为什么它的博客是http://打头的，原来2016年6月后新创立的博客，Github官方才会支持https://,具体通知在这里。

所以目前这个问题，还是很新颖哒！值得解决下~

结论来咯！

综上，要解决这问题，有两个手段：

  1. 买一个域名进行绑定~(岛主你若买，那帮我也买一个吧~😋mua~)
  2. 按照博客二提供的解决方法去尝试


这下应该没问题了吧~继续看书去咯！
