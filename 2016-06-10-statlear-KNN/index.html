关于K-NN算法

从Bayes分类器讲起

page 37 有一个例子

图释如下：

  • 图中有蓝色和橘色的点各100个，是计算机模拟产生的数据

  • 数据是基于我们设定好的随机模型产生的，随机模型给定了平面中任何一个点取蓝色和橘色的概率，例如，$P((1,1)这个点是蓝色)=0.3$&$P((1,1)这个点是橘色)=0.7$，此时$(1,1)$虽然有更大的概率是橘色，但是，蓝色也不是不可能的。

  • 基于上述那个随机模型（它就是真实模型），把平面上蓝色概率和橘色概率都为0.5的点用虚线连起来，这个时候，就有了传说中的三八线。

  • Bayes分类器有着「全对不可能，因为随机性无法对抗」无法对抗，那我就聪明地傻一点「以三八线为『Bayes decision boundary』」，给定一个点，颜色就取为概率大于0.5的那个颜色呗，与真实观测到的不一样也是没办法的，因为只有这样做才总体最优（还有理论证明哦！）

  • 这个时候，再回到这张图，出现在橙色区域的蓝色的点代表什么？代表，这个点有更大的可能性取成橙色，但是最终我们观察到的是蓝色，Bayes分类器出错了。

  • 所幸，出错的点并不多。

The Bayes Classifier produces the lowest possible test error rate, called the Bayes error rate. Since the Bayes classifier will always choose the class for which $Pr(Y=j|X=x_0)$ is largest, the error rate at $X = x0$ will be $1- max{j}Pr(Y = j|X = x0)$. In general, the overall Bayes error rate is given by
$1 - E{max{j}Pr(Y = j |X)}$, where the expectation averages the probability over all possible values of X, because the classes overlap in the true population so $max{j} Pr(Y = j|X = x{0}) < 1$ for some values of $x_0$. The Bayes error rate is analogous to the irreducible error, discussed earlier.

这段话写得很经典，有几处值得再讲一下：

  1. 公式 $Error=1 - E{max_{j}Pr(Y = j |X)}$ 是对$X$求期望，后半部分的期望展开就是

    $ E{max{j}Pr(Y = j |X)} = \int{x \in X} max_{j}Pr(Y = j|X=x)P(X=x)dx$

  2. 多看几眼就知道这个式子很直观，我们对于每个点都取它最有可能取得的值，作为我们给出的预测值，那么在最优可能之外的那些可能性就是我们犯错的可能性。样本空间中的点$x_0$「出现的概率」*「在这点我们『犯错的可能性』」，也可看做是「犯错可能性」的权重。

  3. 例如，如果一个点非常非常容易被取到，那么在这一点上犯错的可能性占整体犯错比重就更大。最后，遍历所有点加在一起。

  4. 总之，任何期望本质不离加权平均。

从Bayes分类器到K-NN

刚培养了点对Bayes分类器的好感，没想到就迎来噩耗——

But for real data, we do not know the conditional distribution of $Y$ given $X$, and so computing the Bayes classifier is impossible. Therefore, the Bayes classifier serves as an unattainable gold standard against which to compare other methods.

在接受了「就是不知道真实的条件分布」这个现实后，没关系，抹把眼泪继续看。

虽然我们不知道真实的条件分布，但是数据中还是有很多信息的，基于数据中给出的信息，可以去估计其条件分布，然后再将一个给定的观测分类到「估计的分布概率」最大的类别中。

此处眼泪应该已风干😉

休息一下吧～

☝️公式

$Pr(Y = k |X = x{0}) = \frac{1}{K}\sum{i \in N{0}} {I(y{i}=j})$

几句解释

  • 在$x_{0}$这一点，我们并不知道$Y$取到各个类别的真实概率，也就无法了解「其在哪个类别的可能性最大」

  • BUT, 我们知道离它最近的$K$个点到底属于哪些类，如果离它最近的哪些点中最大的群体是类别1，好嘞，那我也站到类别1中吧！

Trade-off between Linear Regression and K-NN

page 104

线性回归是参数结构，K-NN是非参结构。参数结构对模型有着很强的假设，如果假设与实际相差很远，则会导致不尽如人意的拟合。

下面👇这张图说明了：K-NN方法选择K时：小的K值提供了较为灵活的拟合但是付出的代价是高方差；大的K值提供的拟合平滑，方差小，其对一个区域的拟合是几个点的均值，所以改变一个点对于结果来说影响不大，但是较大的K值可能会错过真实模型的结构而导致较大的偏差。

在一维的情况下，结果非常符合直观见解（暂时的）：

一维时，真实模型为线性时，线性回归表现灰常好，但是K-NN也不差

下图中👇 红色的点代表由真实的模型「线性」随机生成的点，用这些点中的信息得到拟合直线$\hat{f}(X)$和$K=9, 1$时的K-NN曲线。简而言之，线性回归模型较好，但是K-NN在比较合适的K值下也不差。

下图👇是从MSE来看的，虚线是由最小二乘方法得到的回归直线，实线是真实模型，可以看到，在数据量为100的情况下，当假设与真实相近时，哪怕得到的信息有偏差，但是还是可以越过偏差得到正确的见解。下图👇中的右图👉表示了K-NN与线性回归的MSE，可以看到，合适的K会使得K-NN的MSE与线性回归的MSE相差不多，在这幅图中，估摸了下，粉色线上面的那个点比较合适：）此时的K是3.

一维时，真实模型为非线性时，线性回归表现灰常不好，但是K-NN依旧稳定（这不很显然嘛？）

下图👇中的真实模型（粗粗的黑线）的弯曲程度依次增大，代表真实模型与线性假设渐行渐远。
下图👇中两幅右图👉的黑色虚线对应的MSE分别是0.06与0.14（毛估估哒），也就是说，线性模型在测试集上产生的误差，随着真实模型非线性的增强而不断变大。
下图👇中两幅右图👉的绿色虚线对应的最小MSE分别是0.55与0.05（毛估估哒），也就是说，K-NN在测试集上产生的误差，基本不变。
下图👇中两幅左图👈也很直观滴体现了，K-NN有着「你变弯曲我也变弯曲」的随和品质😉

此处有个大大的BUT

大家自然会觉得，天呐，那我随时用K-NN不就好啦，不管线性还是非线性，它都表现得如此之好！

注意，上述模拟有着非常大的前提「X是一维的变量」，当X的维读增大时，情况就有了惊天的变化！详见下图👇

When p=1 or p=2, KNN outperforms linear regression. But for p=3 the results are mixed, and for p >=4 linear regression is superior to KNN. In fact, the increase in dimension has only caused a small deterioration in linear regression test set MSE, but it has caused more than a ten-fold increase in the MSE for KNN.

简而言之，就是维数灾难，在高维情况下，给定的观测附近没有邻点，与$x_0$最为接近的K个观测值在p维空间中距离$x0$很远，导致其对$f(x{0})$的预测非常差。

举个小栗子吧~每个人都非常非常不一样，如果用很多维度的数据来描述一群人，那么这群人中的每两个都会显著不同（在很多维度都不同），那么用离A最近的K个人的特点取平均来描述A，肯定与A本身差别很大。

不过，穷穷的统计学家的电脑中模拟的结果得出的结论必定是支持线性回归的，但是，真正的海量数据又是另外一回事咯！

As a general rule, parametric methods will tend to outperform non-parametric approaches when there is a small number of observations per predictor.

但还是尝试了一下，将购买与不购买的比例从1:15提升到了1:3，然后用了下统计中re-sample的思想重复抽样100次，得到100个小模型的结果，汇集再得出预测的结论。100次中，如果95次的预测结果是不买，另外5次的预测结果是购买，那么，在利润非常高的情况下，也是值得一试的。总的来说，我做的优化还太少，结果只改善了一点点，还是以撒大网为代价的。

由此，深觉高维的问题果然是个难点。不知道下次先用lasso进行下变量选择再用K-NN加上Boostrap的思想会如何。值得一试的赶脚。
